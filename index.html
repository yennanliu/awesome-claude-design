// 4 steps

Ship a system in one paste

No design theory required. Pick a family, hand the DESIGN.md to Claude, and iterate with inline comments to stay under your token budget.

Pick a family

Read-heavy → Editorial. Scan-heavy data → Data-Dense. Consumer app → Playful. Unsure? Run the family-picker prompt pack.

Copy a DESIGN.md

Open the file in the family folder, copy the whole spec. In a hurry? The last section ("Agent Prompt Guide") is a one-paragraph distillation.

Paste into Claude

Drop it into Claude Design, or save it as DESIGN.md at your repo root for Claude Code. Tell it to obey the colors, type, and component rules.

Iterate with comments

Tweak with inline comments, not fresh chats — vision tokens cost ~3× text. Branch for variants, then hand off to Claude Code in one shot.

claude-code — design-system
# add a design system to your project $ cp design-md/data-dense/crewai.md ./DESIGN.md $ claude "read DESIGN.md, build the dashboard from it. obey colors, type, components." ▸ reading DESIGN.md ok ▸ tokens --bg --accent --text ok ▸ build components/ ok ▸ note no AI-slop fingerprints ✓ shipped 1,284,302 tok · $4.17 · 18s
